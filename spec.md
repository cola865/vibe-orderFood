## Google Sheets 資料結構

| Sheet 名稱 | 欄位 | 用途 |
|---|---|---|
| `Users` | A: 姓名 / B: Email / C: 權限 | 白名單授權 + 角色控制 |
| `Menu` | A: 餐廳 / B: 品項 / C: 價格 / D: 類型 | 完整菜單資料 |
| `TodayConfig` | A: 今日餐廳 / B: 收單狀態(OPEN/CLOSED) | 當日設定 |
| `Orders` | A: 時間 / B: Email / C: 餐廳 / D: 品項 / E: 價格 / F: 備註 | 訂單記錄 |

## 功能清單

### 🔐 認證 / 登入

- Google OAuth 2.0 三方登入（帳號選擇器彈窗）
- **白名單機制**：登入後比對 Google Email 與 `Users!B` 欄，不在名單內顯示「未授權」
- Token 存於 `sessionStorage`，有效期間內重開頁面不需重新登入
- 登出按鈕：清除 sessionStorage，重載頁面

### 🍱 點餐（我要點餐）

- 顯示今日開放的餐廳（由管理員設定）與收單狀態（🟢 收單中 / 🔴 已結單）
- 依餐廳分類顯示菜單卡片
- **一般餐廳**：點擊「點餐」按鈕直接下單
- **滷味類餐廳**（餐廳名稱含「滷」）：Checklist 勾選模式，支援主餐、加購、口味分類，點擊「確認下單」批次送出
- 點餐前即時驗證收單狀態與餐廳是否仍在設定中
- 收單後按鈕 disabled

### 📋 我的餐點

- 顯示目前登入者當日所有訂單
- 以 `#Bxxxx` BatchID 為單位分組顯示（相同批次的品項合併呈現）
- 顯示個人總計金額
- 每筆訂單可刪除（X 按鈕）

### 📊 今日訂單（彙整）

- 顯示今日所有人的訂單
- 以 BatchID 為單位分組，顯示訂單編號（ORD-001 起）、訂購人、品項、總計
- 管理員可刪除任意訂單
- **一鍵複製（LINE 格式）**：產生格式化文字方便貼到 LINE 群通知

### ⚙️ 管理專區（僅管理員）

- 切換收單狀態（OPEN ↔ CLOSED）
- 設定今日開放餐廳（Radio 單選，最多一間）
  - 可選「清空，今日不訂餐」
  - 儲存時自動清除不屬於今日餐廳的歷史訂單
- 🔥 清空今日點餐：刪除 `Orders` 所有記錄 + 清除今日餐廳設定

## 訂單分組邏輯

```
同一批次 = 相同 #Bxxxx BatchID + 相同 Email
→ 合併為一筆「訂單」
→ 相同品項名稱自動累加數量 (x2, x3...)
→ 備註欄顯示加購/口味選項，移除 #Bxxxx 標記
```

---

## 已知限制 / 注意事項

> [!IMPORTANT]
> - 登入狀態只存在 `sessionStorage`，關閉分頁即登出
> - 今日開放餐廳僅能選**一間**（Radio 設計）
> - 清空今日點餐是**不可逆操作**
> - 新用戶首次登入若授權 Token 缺少 `email` scope，需在無痕模式重新授權
